Purpose: capture each verified v9 capability with exact evidence, then compare it to v10 and Bob standards.

This file is the audit ledger. Do not mark a row `Covered` unless the v10 file and evidence column both point to concrete text, diagram, script, export, or live verification.

## 1. Coverage Status Values

| Status | Meaning |
|---|---|
| Covered | v10 explicitly preserves the capability with equal or stronger behavior |
| CoveredWithChange | v10 preserves the intent but renames, moves, or changes implementation |
| Partial | v10 mentions the concept but lacks required design detail |
| Missing | no current v10 coverage found |
| IntentionallyRemoved | removed by design, but requires explicit approval rationale |
| NeedVerify | requires live Fabric, SQL, REST, or git-history verification |

## 2. Bob Alignment Values

| Status | Meaning |
|---|---|
| Aligns | v9/v10 behavior matches Bob standards directly |
| Adapt | Bob standard applies, but must be adapted for Fabric |
| Conflict | Bob standard conflicts with Fabric/v9 design and needs decision |
| NotApplicable | standard does not apply to this object/capability |
| NeedClarification | requires Bob/Rakesh/team decision |
